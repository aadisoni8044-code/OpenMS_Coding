01 — About

What is OpenMS?

OpenMS is a personal side project — a hand-crafted, experimental logic engine written as a learning exercise. It implements a tiny custom syntax that lets you define data and pass it through a simple output mechanism called machine().

This project exists solely to explore how interpreters and language parsers work at a basic level. It is a Proof of Concept (POC) and is openly shared so others can study, fork, and learn from it — nothing more.

02 — Syntax Examples

How OpenMS Looks

The syntax is intentionally small. The core keyword is machine(), which outputs a value. Variables are assigned with =. That's the gist of it.

Example 1 — Direct Output 
// Pass a string directly to the output function
machine("Hello World")
Example 2 — Variable Then Output 
// Assign a value to a variable, then pass it to machine()
data = "Hello World"
machine(data)

03 — How It Works

Under the Hood

OpenMS processes your code through a basic pipeline — nothing fancy, completely transparent.

01

Tokenization

The source text is split into recognizable tokens: keywords, identifiers, strings, and operators.

02

Parsing

Tokens are parsed into a simple structure the engine can evaluate — no complex AST, just the basics.

03

Evaluation

The engine walks the parsed structure, resolves variable references, and executes machine() calls.

04

Output

The result is written to the console. That's it. Simple, honest, and easy to trace.
